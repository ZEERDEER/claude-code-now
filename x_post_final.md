前几个月犯了个错误。

Claude Code Now v1.5.0 加了 API 切换器，结果项目变复杂了，用户也困惑了。

我突然意识到：这违反了我最核心的哲学——简单。

API 切换这事儿，官方的 settings 已经很好用了，社区也有各种工具，我为什么要重复造轮子？

而且加了这功能后，核心价值——"3秒启动"——反而被淹没了。

所以 v1.6.1 我做了个决定：删除 885 行代码，把整个 API 切换器功能都砍掉。

现在 Claude Code Now 只做一件事：让你在 3 秒内启动 Claude Code，不管在哪个文件夹，一键即达。

承认错误不可耻。不承认错误，让产品越走越偏，才可惜。

简单不是减法，是专注。
